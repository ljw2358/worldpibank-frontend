FAQ · 보안

채팅 열기(데모)
지갑 해킹을 어떻게 예방하나요?
• 파이/뱅킹 전용 기기 사용, OS/보안 패치 최신 유지
• 화면 오버레이/알 수 없는 소스 금지, 신뢰된 스토어만 사용
• 임의 QR 스캔 금지(현재 설계상 비활성화)
• 강력한 잠금/생체인증 사용, 시드 문구 공유 금지
• 의심되면 새 지갑으로 마이그레이션·세션 회수·키 회전·체인 탐색기 모니터링
휴대폰이 해킹되면?
• QR 흐름 중단, 신뢰 기기에서만 작업
• 모든 비밀번호 교체, 필요 시 백업 후 공장초기화
• 새 지갑 생성 → 소액 테스트 이체 → TXID 기록 보관
KYC 화면 없이 어떻게 검증하나요?
**단계적 검증**

1) 경량 점검: 기기 평판, IP 리스크, 중복계정/속도/제재 리스트 이름 스크리닝
2) 문서+라이브니스(조건부): 리스크 임계 초과 시 안전한 플로우로 요청
3) 주기적 재검증: 리스크 기반 주기, 암호화 저장, 최소권한·기한 기반 접근
어떤 데이터를 저장하나요?
컴플라이언스 최소치: 해시된 식별자, 리스크 점수, 감사 로그. 민감 문서는 암호화 저장·최소권한 접근.
왜 영어가 기본인가요?
글로벌 기본은 영어이나, 상단에서 한국어(KR)/중국어(中文)로 전환 가능합니다.
해커톤에서 무엇이 중요하죠?
명확한 UX, 동작 데모, 보안 설계, 문서화된 흐름, 빠른 온보딩.
