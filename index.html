How do I protect my wallet from hacking?
• Use a dedicated device for Pi and banking, with the latest OS and security patches.
• Disable screen overlays & unknown sources; install apps only from trusted stores.
• Avoid scanning arbitrary QR codes (feature is currently disabled by design).
• Use strong, unique passcodes; enable biometric lock. Never share seed phrases.
• On suspected compromise: migrate to a new wallet, revoke device sessions, rotate keys, and monitor chain explorers.
What if my phone was hacked?
• Immediately stop using QR flows; operate via a clean device.
• Change all credentials from a trusted machine; backup & factory-reset the old device if needed.
• Create a fresh wallet; move funds in small test transfers; keep audit notes of TXIDs.
How will KYC be verified without a KYC screen?
We don’t show a KYC screen here. Instead, verification follows a **phased approach**:

1) **Lightweight checks**: device reputation, IP risk, duplicate account heuristics, velocity & sanctions name screening.
2) **Document + liveness (on-demand)**: when risk thresholds trigger, we request ID scan & selfie liveness via a secure flow (modular, replaceable provider).
3) **Periodic re-verification**: risk-based cadence; privacy-preserving data retention and deletion schedule.

Result: users pass with minimal friction while high-risk sessions receive deeper checks.
What data is stored?
Only what’s necessary for compliance: hashed identifiers, risk scores, and audit logs. Sensitive documents are encrypted at rest; access is least-privileged and time-bounded.
